Question 3

Class Components vs Function Components in React

React components can be created using either Class Components or Function Components. Both serve the same purpose but have different syntax and features.

Aspect Class Components Function Components
Syntax ES6 class extending React.Component JavaScript function returning JSX
State Management this.state and this.setState() useState() hook
Lifecycle Methods componentDidMount, componentDidUpdate, etc. useEffect() hook
Code Length More verbose More concise
Performance Slightly slower Slightly faster

Class Component Example:

import React from 'react'; class Welcome extends React.Component { constructor(props) { super(props); this.state = { count: 0, message: 'Hello from Class Component' }; } incrementCount = () => { this.setState({ count: this.state.count + 1 }); } componentDidMount() { console.log('Component mounted'); } render() { return ( <div> <h1>{this.state.message}</h1> <p>Count: {this.state.count}</p> <button onClick={this.incrementCount}> Increment </button> </div> ); } } export default Welcome;

Function Component Example (with Hooks):

import React, { useState, useEffect } from 'react'; function Welcome() { const [count, setCount] = useState(0); const [message, setMessage] = useState('Hello from Function Component'); const incrementCount = () => { setCount(count + 1); }; useEffect(() => { console.log('Component mounted'); }, []); return ( <div> <h1>{message}</h1> <p>Count: {count}</p> <button onClick={incrementCount}> Increment </button> </div> ); } export default Welcome;

Key Differences:

  • Function components are simpler and more concise
  • Class components use this keyword, function components don't
  • Function components with hooks are the modern React approach
  • Both can achieve the same functionality

Using Lists to Display Student Marks

In React, we can use the map() function to render lists of data. Here's how to display marks for five students:

Example: Student Marks Component

import React from 'react'; function StudentMarks() { // Array of student objects with marks const students = [ { id: 1, name: "Alice", marks: 85 }, { id: 2, name: "Bob", marks: 92 }, { id: 3, name: "Charlie", marks: 78 }, { id: 4, name: "Diana", marks: 95 }, { id: 5, name: "Eve", marks: 88 } ]; return ( <div> <h2>Student Marks List</h2> <ul> {students.map((student) => ( <li key={student.id}> {student.name}: {student.marks} marks </li> ))} </ul> </div> ); } export default StudentMarks;

Advanced Example with Table:

import React from 'react'; function StudentMarksTable() { const students = [ { id: 1, name: "Alice", subject: "Math", marks: 85, grade: "A" }, { id: 2, name: "Bob", subject: "Science", marks: 92, grade: "A+" }, { id: 3, name: "Charlie", subject: "English", marks: 78, grade: "B" }, { id: 4, name: "Diana", subject: "History", marks: 95, grade: "A+" }, { id: 5, name: "Eve", subject: "Geography", marks: 88, grade: "A" } ]; return ( <div> <h2>Student Performance Report</h2> <table> <thead> <tr> <th>ID</th> <th>Name</th> <th>Subject</th> <th>Marks</th> <th>Grade</th> </tr> </thead> <tbody> {students.map((student) => ( <tr key={student.id}> <td>{student.id}</td> <td>{student.name}</td> <td>{student.subject}</td> <td>{student.marks}</td> <td>{student.grade}</td> </tr> ))} </tbody> </table> </div> ); } export default StudentMarksTable;

Key Points:

  • map() function: Transforms each item in an array into JSX elements
  • key prop: Required for list items to help React identify changes efficiently
  • key should be unique: Use IDs or unique identifiers, not array indices
  • Arrow functions: Provide concise syntax for mapping

Routing in ReactJS

React Router is a standard library for routing in React applications. It enables navigation between different views/components without page refresh.

Installation:

# Using npm npm install react-router-dom # Using yarn yarn add react-router-dom

Basic Routing Example:

import React from 'react'; import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'; // Component definitions function Home() { return <h2>Home Page</h2>; } function About() { return <h2>About Page</h2>; } function Contact() { return <h2>Contact Page</h2>; } function App() { return ( <BrowserRouter> <nav> <ul> <li><Link to="/">Home</Link></li> <li><Link to="/about">About</Link></li> <li><Link to="/contact">Contact</Link></li> </ul> </nav> <Routes> <Route path="/" element={<Home />} /> <Route path="/about" element={<About />} /> <Route path="/contact" element={<Contact />} /> </Routes> </BrowserRouter> ); } export default App;

Advanced Example with URL Parameters:

import React from 'react'; import { BrowserRouter, Routes, Route, Link, useParams, useNavigate } from 'react-router-dom'; // Component to display user profile function UserProfile() { const { userId } = useParams(); return ( <div> <h2>User Profile</h2> <p>User ID: {userId}</p> </div> ); } // Component with programmatic navigation function Dashboard() { const navigate = useNavigate(); const goToProfile = () => { navigate('/user/123'); }; return ( <div> <h2>Dashboard</h2> <button onClick={goToProfile}> Go to Profile </button> </div> ); } function App() { return ( <BrowserRouter> <nav> <Link to="/">Home</Link> | <Link to="/dashboard">Dashboard</Link> | <Link to="/user/101">User 101</Link> </nav> <Routes> <Route path="/" element={<Home />} /> <Route path="/dashboard" element={<Dashboard />} /> <Route path="/user/:userId" element={<UserProfile />} /> <Route path="*" element={<NotFound />} /> </Routes> </BrowserRouter> ); } function NotFound() { return <h2>404 - Page Not Found</h2>; }

Key Concepts:

  • BrowserRouter: Provides routing context for the entire application
  • Routes: Container for all Route components
  • Route: Defines a path and the component to render
  • Link: Creates navigation links without page refresh
  • useParams: Hook to access URL parameters
  • useNavigate: Hook for programmatic navigation
  • Dynamic Routes: Use :parameter syntax for URL parameters
  • 404 Route: Use path="*" as a catch-all for undefined routes
